## 1. 训练系统设计

### Q1: 如何设计分布式训练系统

**答**：
```
┌─────────────────────────────────────────────────────────────┐
│                      分布式训练架构                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Coordinator/Scheduler                                      │
│       ↓                                                     │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐                     │
│  │ Worker 1│  │ Worker 2│  │ Worker N│  ← 数据并行          │
│  └────┬────┘  └────┬────┘  └────┬────┘                     │
│       ↓            ↓            ↓                           │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐                     │
│  │ GPU 1   │  │ GPU 2   │  │ GPU N   │  ← 模型并行         │
│  └─────────┘  └─────────┘  └─────────┘                     │
│       ↓            ↓            ↓                           │
│  ┌─────────────────────────────────────────┐               │
│  │         AllReduce / NCCL 通信              │               │
│  └─────────────────────────────────────────┘               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**关键组件**：
- 数据分片和加载
- 梯度同步（AllReduce）
- 模型分片（ZeRO/Pipeline）
- 容错和恢复

## 2. 推理系统设计

### Q2: 如何设计高并发推理服务

**答**：
```
┌─────────────────────────────────────────────────────────────┐
│                      推理服务架构                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│                    Load Balancer                            │
│                         ↓                                   │
│  ┌──────────────────────────────────────────────┐          │
│  │              API Server (FastAPI)              │          │
│  └──────────────────────────────────────────────┘          │
│              ↓                    ↓                         │
│  ┌────────────────┐    ┌────────────────┐                    │
│  │   vLLM实例1   │    │   vLLM实例2   │    ← 水平扩展      │
│  └────────────────┘    └────────────────┘                    │
│              ↓                    ↓                         │
│  ┌─────────────────────────────────────────┐               │
│  │           GPU Cluster                    │               │
│  │  ┌─────┐  ┌─────┐  ┌─────┐  ┌─────┐  │               │
│  │  │GPU 1│  │GPU 2│  │GPU 3│  │GPU 4│  │               │
│  │  └─────┘  └─────┘  └─────┘  └─────┘  │               │
│  └─────────────────────────────────────────┘               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**优化策略**：
- Continuous Batching
- PagedAttention/KV Cache
- 量化（INT8/FP16）
- 异步处理

## 3. RAG系统设计

### Q3: 如何设计企业级RAG系统

**答**：
```
┌─────────────────────────────────────────────────────────────┐
│                     企业级RAG架构                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  数据层                                                     │
│  ├── 文档存储 (S3/MinIO)                                    │
│  ├── 向量数据库 (Milvus/Pinecone)                           │
│  └── 关系数据库 (PostgreSQL)                                │
│       ↓                                                     │
│  服务层                                                     │
│  ├── 文档处理服务 (PyMuPDF/Docling)                         │
│  ├── 向量化服务 (Embedding Model)                           │
│  ├── 检索服务 (Hybrid Search)                              │
│  └── 生成服务 (LLM)                                          │
│       ↓                                                     │
│  应用层                                                     │
│  ├── API Gateway                                            │
│  ├── 监控系统 (Prometheus/Grafana)                         │
│  └── 缓存层 (Redis)                                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
